## Overview

Automatically locks and unlocks a SESAME smart lock at scheduled times. A DS1302 RTC keeps accurate time, and ESP32C6 deep sleep minimizes power consumption. A serial shell at boot lets you configure the time and schedule interactively.

| --- | --- | --- |

## Operation Flow

```text
Boot
  │
  ├─[Fresh boot]──→ Serial shell (10-second window)
  │                    ↓ "run" or timeout
  └─[Wake from deep sleep]
        │
        ↓
    Check current time via RTC
        │
        ├─[Unlock time passed, not yet unlocked]
        │     → BLE scan → connect → unlock()
        │
        ├─[Lock time passed, not yet locked]
        │     → BLE scan → connect → lock()
        │
        └─[Not yet time]
              → Deep sleep until next event (max 1 h)
```

## Environment Setup

### Requirements

- [VS Code](https://code.visualstudio.com/)
- [PlatformIO IDE extension](https://marketplace.visualstudio.com/items?itemName=platformio.platformio-ide)
- USB-C cable (for connecting the XIAO)

### Setup Steps

1. Install the **PlatformIO IDE** extension in VS Code
2. Clone this repository

   ```bash
   git clone https://github.com/zhixuan2333/sesami-lock-auto-lock-unlock.git
   cd sesami-lock-auto-lock-unlock
   ```

3. Open the folder in VS Code — PlatformIO will automatically fetch libraries
4. Configure your Sesame keys

   ```bash
   cp src/mysesame-config.h.example src/mysesame-config.h
   # Edit SESAME_SECRET / SESAME_PK / SESAME_MODEL
   ```

5. Click **Upload** in the PlatformIO sidebar to build and flash
6. Open **Serial Monitor** to view output (115200 baud)

## Getting SESAME Keys

1. Show the device QR code in the SESAME app
2. Scan it at [https://sesame-qr-reader.vercel.app/](https://sesame-qr-reader.vercel.app/)
3. Copy `Secret Key` and `Public Key` into `mysesame-config.h`

```cpp
// src/mysesame-config.h
#define SESAME_SECRET "your_secret_key_here"
#define SESAME_PK     "your_public_key_here"
#define SESAME_MODEL  Sesame::model_t::sesame_5_pro  // change to your model
```

## Serial Shell Commands

After boot, a 10-second window opens for serial commands (115200 baud).

```text
> help
Commands:
  time                         - show current RTC time
  time set YYYY-MM-DD HH:MM:SS - set RTC time
  schedule                     - show unlock/lock schedule
  unlock HH:MM                 - set daily unlock time (default 08:00)
  lock HH:MM                   - set daily lock time   (default 20:00)
  reset                        - reset today's action flags
  run                          - exit shell and start operation
```

### Example

```text
> time set 2026-05-08 09:00:00
> unlock 07:30
> lock 22:00
> run
```
